# Contributing

Thanks for considering a contribution to OSINT Toolkit Local.

## Scope

This project only accepts legal and ethical OSINT functionality:

- Public data only
- No brute force
- No authentication bypass
- No exploit attempts
- No credential collection
- No aggressive scanning

## Development Workflow

1. Open an issue describing the proposed change.
2. Keep changes focused and scoped to one concern.
3. Update documentation when behavior, setup, architecture, or workflow changes.
4. Run backend and frontend checks before opening a pull request.

## Local Checks

```bash
cd backend
dotnet test
```

```bash
cd frontend
npm run build
```

## Pull Requests

Include:

- Summary of the change
- Testing performed
- Documentation updated
- Any ethical or legal considerations
